# INV-0b 合并幻觉 (Merge Hallucination)

**分类**: 已确认 invariant
**Target skill**: `harness-eng`

## 模式定义

多个并行 WP 完成后，集成者**口头**宣布"已合并"，但实际的文件合并动作要么未做、要么只做了文本级拼接而忽略了语义冲突。集成者的"已合并"状态与文件系统状态不一致。

INV-0（快照幻觉）是决策 vs 执行的时间差；INV-0b 是"集成完成"这一状态本身被幻觉化。

## 典型形状

- "A 的 PR 和 B 的 PR 都 green，合进来就行"——实际 A 改了某个函数签名，B 在旧签名上加了新调用。
- 集成 agent 报告 "3 个 WP 已集成"，但 `git log` 显示只有 2 个 merge commit，第三个还在并行分支。
- 文本 diff 看起来干净，但 A/B 都改了同一个配置项的不同字段，语义互相覆盖。

## 检测信号

- "已合并"声明与 `git log --merges` 不匹配
- 集成后测试跳过或被标 BLOCKED 却未触发回滚
- 集成文档提到"无冲突"但 diff 中有同文件多段互不相关的修改

## 缓解动作

**注入到 `harness-eng`**：

1. "集成完成" = `git merge` 成功 + 回归测试通过 + 集成者在 LOG 中给出 `git log --oneline -n 5` 证据。三者任一缺失 → 集成未完成。
2. 并行 track 在合流前必须走一次**语义级 diff review**，不能只看文本 diff。
3. 集成 owner 不等于并行 WP 的任一执行者（避免自证）。

## 与 INV-0 的关系

INV-0 防的是"决策时的快照已过时"。INV-0b 防的是"执行结果的快照（即 agent 脑中的集成状态）与真实状态已分叉"。两者都是"脑中的世界 vs 文件系统的世界"脱节，但发生在状态机的不同位置。
