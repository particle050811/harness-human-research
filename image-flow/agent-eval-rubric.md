# image-flow 评测验收标准（评分细则）

> 本文档供**评测者**使用，与 `agent-eval-spec.md`（开发需求文档）配套：把需求文档作为被测 AI Agent 的唯一输入，实现完成后按本文档逐项打分。**不要把本文档发给被测 Agent**。条款中的 § 编号均指需求文档章节。

## 评分规则（满分 100）

- 每个小项**独立计分**：做到即得分，与其他小项是否完成无关；某项只做对一半（如解析对了但替换编号错）按该项 0 分计，不再细分。
- 后续阶段不依赖前面阶段满分——例如 M2 没做侧栏，M3 的异步机制仍可通过右键命令验证计分。
- 任一阶段验收前提是 `npm run compile` 可通过（见 M1.1）；编译不过则整体只能得 M1.1 之前已得的分。

## 端到端冒烟清单（完成度初筛）

逐项打分用上面的评分细则；以下 12 条快速冒烟检查用于初筛，全过则大概率拿到高分：

1. `npm run compile` 零 error 零 warning 通过（含两套类型检查与 lint）。
2. F5 启动后：活动栏出现图标，侧栏三个标签页可用，配置改动即存、重载窗口后仍在。
3. apiKey 不出现在 globalState 中（存 secrets）。
4. 对含两张参考图（其中一张路径带空格用 `<>` 包裹）的 MD 点「预览请求」：prompt 中图片语法被替换为 `[image1]`/`[image2]`，参考图概览两条，请求参数无 prompt/images 字段。
5. 并发数设 3 点生成：立即出现进行中卡片（提交中 3/3），按钮几乎不阻塞；卡片展示进度条与逐秒计时；完成后图片落在同一 `task-*` 文件夹且文件名连续编号不覆盖。
6. 生成进行中重启 VS Code（或重载窗口）：任务卡片恢复并继续轮询直到完成。
7. 任务全部失败：弹错误通知、卡片消失、空 task 文件夹被清理。
8. 历史页正确列出既有 task-* 文件夹，进行中任务不重复出现。
9. 素材库：手动添加文件夹可递归显示图片；打开子目录中的 MD 时「当前路径」逐层出现自动库；右键缩略图把 `![alt](./相对路径)` 插入光标处，含空格路径自动加 `<>`；活动编辑器不是生效 MD 时拒绝插入。
10. 切换模型为 `gpt-image-2-vip` 后预览请求中 `aspectRatio` 为像素值（如 `864x1152`）。
11. 工作区根放置 IMAGES.md 后，预览的 prompt 前部包含其全文与模型注入句。
12. 单元测试全部通过。
