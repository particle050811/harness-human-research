# empty 运行时人工测试

被测变体：empty（无 skill 基线），run `eval-manual-supervised-empty-260611160043`。
驱动方式：监工模式逐轮验收。本文只记 F5 扩展开发宿主里手动操作发现的运行时缺陷。

## 缺陷一：工作台素材库未做高度限制，参数区被挤到底

**现象**：工作台 Tab 下「素材库」展开后，缩略图按真实条目数一路平铺（参考素材有 43 条人设），列表不设最大高度、不内部滚动，直接把下方的参数行（模型 / 分辨率 / 比例 / 并发数）和「生成」按钮顶到面板最底部，素材多时需滚动整个侧栏才能够到生成。

**复现**：见 `工作台素材库挤压1.png`、`工作台素材库挤压2.png`——对着真实漫画素材目录打开工作台，素材库一展开，常用的参数与生成入口就被推出首屏。

**问题**：参数区与生成按钮是工作台的高频操作落点，应常驻可见；素材库是浏览区，理应自身限高内部滚动，而不是无限撑开把主操作挤走。空状态下（参考图1，素材库折叠）布局正常，正说明这是「列表长度不受控」而非整体布局问题。

**对应 spec**：素材库/工作台布局相关章节——长列表的容器约束属于红线外的体验缺陷，静态四检不暴露，只有运行时受验能发现。
