# wow-harness 运行时人工测试

被测变体：wow-harness，run `eval-manual-supervised-wow-harness-260612165749`。
驱动方式：监工模式逐轮验收。本文只记 F5 扩展开发宿主里手动操作发现的运行时缺陷。

## 侧栏曾白板、起不来（已修复）

初次拉起时侧栏渲染不出（白板），「AI 绘图」标签下整片空白、进不到任何界面（见 `侧栏白板渲染不出.png`）。追加一轮监工反馈后，被测会话定位为 webview 的 React 根组件从未挂载（HTML 容器 `<div id="root">` 始终为空），补上挂载调用后侧栏可正常渲染。下列缺陷均为侧栏能正常进入后实测发现。

## 缺陷一：缩略图每行张数调参失效

**现象**：缩略图每行张数不受设置控制，实际按容器宽度尽量塞满（一行可排下约 11 张），调「每行张数」设置不生效。

**复现**：见 `缩略图每行张数调参失效.png`——截图中「当前路径 分镜\4」一行只显示 5 张，是因为该目录仅 5 张图（并非每行上限）；目录图多时一行会一直塞到约 11 张，始终不跟随「每行张数」设置。

**对应 spec**：`workbenchCols`（工作台素材缩略图每行张数，默认 4、范围 1~8）、`tasksCols`（任务页缩略图每行张数，默认 2、范围 1~8）——这两个参数应实时控制每行张数。

## 缺陷二：设置里点「获取 api key」插件崩溃

**现象**：设置页中点击「获取 api key」后插件直接崩溃（预期打开对应获取页面/链接，实际整个插件挂掉）。
