# gstack 运行时人工测试

被测变体：gstack，run `eval-manual-supervised-gstack-260612144918`。
驱动方式：监工模式逐轮验收。本文只记 F5 扩展开发宿主里手动操作发现的运行时缺陷。

## 备注：初次「侧栏一直转圈、加载不出」非变体缺陷

首次拉起时侧栏卡在 loading 进不去。排查后确认是 run 里**提交的构建产物陈旧**（`dist/extension.js` / `media/sidebar.js` 与源码不一致）导致，**与 gstack 变体本身无关**。

- 复现/修复：`rm -f dist/extension.js && npm run compile` 干净重编译（check-types / lint / esbuild 全过）后重新拉起即正常。
- 工具侧成因：`eval-test.sh` 只用 `[ -f dist/extension.js ]` 判断是否需要构建，存在即跳过，不校验产物是否比源码新——陈旧产物不会被重建。此条不计入 gstack 评分。

## 缺陷

- 重编译后界面能进，但实际使用中仍存在加载不出来的情况（具体表现待复现）。本轮先搁置，下次再查根因。
